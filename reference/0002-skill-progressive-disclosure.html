Skills 渐进式披露速查

把 skill 想成三层上下文：索引用于发现，正文用于执行，附属文件用于当前任务的细节。

1. 索引 name、description、path 进入模型上下文，用于路由。
2. 命中 用户点名 skill，或任务明显匹配 skill 描述。
3. 正文 读取匹配 skill 的 SKILL.md，必须读到 EOF。
4. 解析 相对路径以 SKILL.md 所在目录为基准。
5. 附件 只读取当前任务需要的 reference、scripts、assets、templates。
6. 执行 用展开后的局部上下文完成任务，并保留验证结果。

关键规则

  • 不要把所有下载的 skill 都当成 active skill。
  • active skill 的轻量索引可以常驻；完整正文按需读取。
  • 用户显式点名 skill 时，必须使用该 skill，并完整读取 SKILL.md。
  • SKILL.md 引用的相对路径，先按 skill 目录解析。
  • 附属文件不是全量展开；只读取和当前任务相关的文件。
  • 脚本和模板存在时，优先复用，而不是手写一份相似实现。

一句话模型

Progressive disclosure = catalogue first, selected instruction next, task-specific materials last.

和 Sandcastle 的对应关系

Skill catalogue

像候选 repo 列表：足够 planner 做选择，但还不是完整执行上下文。

Selected SKILL.md

像被批准的 repo issue：一旦命中，就进入可执行说明。

Referenced files

像 executor 的局部材料：按任务读取，避免上下文互相污染。
