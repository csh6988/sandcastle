# Persist current state with append-only audit and a RuntimeEvent outbox

Sandcastle v1.0 does not use full event sourcing. Each Company Runtime command updates queryable current-state tables, appends an immutable audit record, and enqueues protocol-neutral RuntimeEvent records in one SQLite transaction; an outbox publisher then delivers those events to Desktop AG-UI sessions, ACP sessions, and observability consumers with durable cursors and replay after disconnection. RuntimeEvent remains the single structured observation model but is not the sole database source of truth, and adapter delivery failures cannot roll back completed workflow work. Run recovery uses immutable configuration snapshots and node checkpoints rather than replaying the complete UI event history. This model is chosen over full event sourcing because v1 needs transactions, auditability, and resumable streams without making every query and migration depend on event replay.
