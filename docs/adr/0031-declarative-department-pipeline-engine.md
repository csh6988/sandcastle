# Department Pipelines use a declarative DAG engine over existing execution primitives

Sandcastle v1.0 adds a declarative **Department Pipeline Engine** that owns versioned DAG validation, dependency scheduling, serial and parallel execution, conditions, human approvals, recovery, and Artifact contracts. It exposes a closed v1 node set—Start, AI Task, Human Approval, Condition, Parallel/Join, and Complete—and delegates work through injected node handlers; arbitrary user-authored code nodes are excluded. Existing Agent Provider, Sandbox, Worktree, commit, verification, and selected Board execution capabilities are reused behind production execution adapters, while the fixed Board workflow becomes reference material for the built-in Software Development Department template rather than the new pipeline engine itself. This additional orchestration layer is chosen because the fixed Board workflow cannot safely represent editable, versioned Department DAGs without coupling the new product model to legacy stages.
