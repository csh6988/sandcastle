# The v1.0 company model starts without legacy data migration

Sandcastle v1.0 is a new product data model and does not import, dual-write, or maintain compatibility with fixed-stage Desktop Project records or existing Board task data. Users create a new Company Directory and all v1 state is written only through the new Company Runtime, SQLite schema, Department Pipeline Engine, and Artifact registry; old directories and files remain untouched and can be opened only by the historical product if needed. Existing Agent, Sandbox, Worktree, RuntimeEvent, verification, and selected Board execution code may be reused as implementation primitives, but legacy stores and UI stages are not contracts for the new model. This clean start is chosen to avoid carrying conflicting state semantics into the AI software company architecture.
