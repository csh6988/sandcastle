# A supervised Company Runtime is the single local control plane

Sandcastle v1.0 introduces a dedicated local **Company Runtime** process supervised by Electron. It is the single writer for the Company Directory SQLite database and owns application commands, Department Pipeline scheduling, Run Configuration Snapshots, approvals, permissions, RuntimeEvent publication, Agent/Sandbox worker supervision, and interaction sessions; the Electron main process owns only native shell concerns and process lifecycle, while the React renderer communicates through a narrow local IPC contract and never accesses the database, filesystem, or Agent processes directly. The ACP stdio facade connects to this same runtime instead of creating another state source. This extra process boundary is chosen over hosting orchestration in Electron main because renderer reloads must not interrupt runs, agent work must not block the desktop shell, and future multi-Agent Discussion Topics need one coordinator and event authority.
