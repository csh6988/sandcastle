# Parallel repository nodes require isolated workspaces

Software Development Department nodes that execute concurrently against the same repository must use separate Worktrees or isolated Sandbox workspaces, with independent branches, sessions, checkpoints, and artifact outputs. Join and Review nodes consume commits, patches, and registered Artifact Versions rather than a shared mutable directory, and only dedicated merge handling may update the target branch. Bind-mount and no-sandbox providers remain supported, but same-repository execution is serialized by default unless an explicit safety check permits concurrency. The selected sandbox provider and `head`, `branch`, or `merge-to-head` branch strategy are captured in the Run Configuration Snapshot and cannot change silently during a run. This restriction preserves existing provider compatibility while preventing parallel AI Members from corrupting each other's work.
