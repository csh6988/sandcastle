# Department Runs use immutable configuration snapshot revisions

Every Department Run starts from an immutable **Run Configuration Snapshot** containing the published Pipeline Version, resolved node definitions, Position and AI Member references, Skill Flows, input Artifact Versions, permission policy, and an Execution Profile with provider/model, Sandbox, branch strategy, and resource limits but no credentials or raw secrets. AI Member identity remains independent from that execution profile. A recovery that changes an allowed execution setting creates a new snapshot revision linked to its parent instead of mutating the original; every Node Run, audit record, and produced Artifact Version records the exact revision it used. Forking creates a new Run from a chosen revision and shares no mutable run state. This revision model preserves reproducibility and provenance while still allowing explicit recovery-time provider or sandbox changes.
