# Desktop company state uses SQLite with file-backed artifacts

Sandcastle v1.0 stores authoritative company, department, pipeline, snapshot, run, approval, interaction-session, runtime-event index, and artifact-lineage metadata in a SQLite database inside the selected Company Directory, while artifact contents, source trees, exported evidence, and immutable snapshot payloads remain ordinary files. The SQLite adapter belongs to the Desktop/company-runtime boundary and is not added to the published root npm package. ADR 0022's dependency-free JSON/NDJSON BoardStore remains valid for the historical Board product, but the new company model neither imports nor dual-writes that store; reused Board execution behavior is reached through execution adapters only. This hybrid model is chosen over all-JSON storage because transactions, crash recovery, concurrent sessions, indexed queries, and lineage consistency are core v1 requirements, while keeping deliverable contents as files preserves local inspectability and portability.
