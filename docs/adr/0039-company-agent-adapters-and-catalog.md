# Company Agent Adapters and a Runtime-owned Agent Catalog

The Desktop Company Runtime treats locally installed agents as formally registered **Company Agent Adapters**, identified by stable IDs rather than display names or models. The Runtime owns an Agent Catalog read model containing detection results and exposes a minimal non-destructive test command; adapters never create Runs, mutate repositories, or persist credentials and sensitive command output. A Position owns one default adapter reference, which is snapshotted for each Department Run and can be explicitly overridden only through the existing immutable snapshot/recovery rules; AI member identity remains independent. Skills remain company-wide references discovered from external `SKILL.md` sources, so the Runtime stores metadata and fingerprints rather than copying source contents.
