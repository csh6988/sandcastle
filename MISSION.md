# Mission: Sandcastle multi-repository delivery flow

## Why

Learn how this project turns a product request or PRD into coordinated repository changes, branches, commits, and verification results. The practical goal is to reason about, extend, or operate Sandcastle's multi-repository workflow without losing track of ownership between planning, execution, sandboxing, and delivery validation.

## Success looks like

- Explain when to use `run()`, `runWorkspace()`, and `runWorkspaceTask()`.
- Trace a PRD from input through alignment, technical plan, repository issues, execution, commits, and result reporting.
- Identify where to add safeguards for multi-project delivery verification.

## Constraints

- Ground lessons in this repository's actual code and terminology.
- Keep each lesson short enough to review quickly.
- Prefer diagrams, checklists, and retrieval practice over broad narration.

## Out of scope

- Rewriting the workflow implementation during teaching sessions.
- General AI-agent orchestration theory not needed to understand this project.
